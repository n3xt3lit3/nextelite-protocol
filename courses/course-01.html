LESSON 01
What is Claude?

Claude is an AI assistant built by Anthropic, a company focused on AI safety. Unlike generic chatbots, Claude is designed to be helpful, harmless, and honest — trained with constitutional AI principles that make it reason carefully before responding.

Claude comes in a family of models, each optimized for different use cases. Haiku is the fastest and most cost-effective — ideal for quick tasks and high-volume work. Sonnet balances speed and intelligence — the everyday workhorse. Opus is the most powerful — deep reasoning, complex analysis, creative work at the highest level.

A critical concept is the context window — how much information Claude can hold in a single conversation. Standard models support 200K tokens (roughly a full novel). Opus with extended thinking supports up to 1M tokens — enough to hold an entire codebase, multiple documents, and long conversation history simultaneously.

Claude is the AI behind the NextElite protocol. Every page on this site, every line of code, every design decision — built in conversation between a human and an AI. No templates. No agencies. Just direct collaboration with Claude.

LESSON 02
Prompting Fundamentals

The quality of what Claude gives you depends entirely on what you give it. A vague prompt returns a vague response. A specific, contextualized prompt returns something you can actually use.

The fundamentals: Be specific about what you want. Instead of "write me some code," say "write a JavaScript function that toggles a dark/light theme using a data-theme attribute on the html element." Give context — tell Claude what the project is, what stack you're using, what style you prefer. Use examples — show it what good output looks like.

System prompts set the behavior for an entire conversation. They define Claude's role, tone, constraints, and knowledge base — like a briefing before the mission starts. User prompts are your individual messages within that context.

Temperature controls creativity vs consistency. Low temperature (0.0 - 0.3) gives precise, deterministic answers — ideal for code and factual queries. High temperature (0.7 - 1.0) produces more varied, creative responses — better for brainstorming and writing. Default is usually around 1.0.

LESSON 03
Multi-turn Conversations

Claude remembers everything within a conversation. Each message builds on the last, creating a shared context that grows more powerful with every turn. This is where the real work happens — not in single prompts, but in iterative dialogue.

Building on previous responses is the core skill. Ask Claude to generate something, then refine it. "Make it more concise." "Change the tone to match our brand." "Now add a hover animation." Each instruction layers on top of what came before.

Know when to start fresh vs continue. If the conversation has drifted off-topic or accumulated confusion, a new conversation with a clear system prompt will be more efficient than trying to correct course. If you're deep in a productive flow, keep going — context is valuable.

Managing long conversations matters because even with large context windows, signal-to-noise ratio degrades over time. Periodically summarize where you are. Restate key decisions. Remove ambiguity. Think of it as maintaining a clean workspace — the cleaner the context, the better the output.

LESSON 04
Working with Code

Claude can read, write, debug, and explain code across virtually every programming language. It understands not just syntax but architecture, patterns, performance implications, and best practices. It's a collaborator, not a copy-paste machine.

Sharing code effectively: Give Claude the full file or relevant section, not just a snippet without context. Specify the language and framework. Describe what the code should do and what it currently does (if different). The more context, the better the fix.

Asking for modifications: Be precise. "Add a 300ms ease transition to the nav background" is infinitely better than "make the nav smoother." Point to specific selectors, functions, or lines. Claude works best when the target is clear.

Understanding diffs: When Claude shows you changes, read them carefully. The diff format shows what's removed (old) and what's added (new). This is how you learn — not just from the result, but from understanding what changed and why.

LESSON 05
Images, PDFs, and Files

Claude has vision capabilities — it can analyze images, read documents, process screenshots, and extract information from visual content. This transforms how you work with design, documentation, and reference material.

Image analysis: Share a screenshot of a design you like, and Claude can describe the layout, colors, typography, and spacing. Share a wireframe, and it can generate code. Share a bug screenshot, and it can identify the issue.

PDF and document processing: Claude can read through PDFs, extract key information, summarize findings, and answer questions about the content. Research papers, contracts, documentation — all fair game.

Using vision effectively: Be specific about what you want extracted or analyzed. "Describe the color palette and typography" is better than "what do you see?" Combine image analysis with text instructions for powerful results — "here's the current design, make it match this reference."

LESSON 06
Best Practices & What's Next

Common mistakes to avoid: Vague prompts that give Claude nothing to work with. Not iterating — accepting the first response instead of refining it. Not giving context — Claude can't read your mind, but it can work with everything you share. Trying to do too much in one prompt instead of building incrementally.

Power user tips: Always start with a strong system prompt that defines the project. Save good prompts as templates. Use Claude to review its own output — "what could be better about this?" Chain conversations strategically — use output from one as input to the next. Document your learnings in a living file.

What's next: This course covered Claude as a conversational AI. But Claude goes much further. The API lets you integrate Claude into your own applications programmatically. Claude Code is an agentic coding tool that reads your codebase, runs commands, and builds alongside you in the terminal. MCP (Model Context Protocol) connects Claude to external tools, databases, and services.

The NextElite protocol was built entirely with Claude Code. Every page, every animation, every decision — a human-AI collaboration running in the terminal. That's the next level.
